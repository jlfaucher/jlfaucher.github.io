78 // Guide to adding new classes to ooRexx
79 // 1) Determine the type of class. We have three categories,
80 // a) exported classes (classes visible to the Rexx programmer), b)
81 // external classes (classes that are not directly visible to the
82 // Rexx programmer, but can end up in a saved image or in a program
83 // compiled by rexxc, and c) transient classes that never get saved, but
84 // manage the run time state of the program. All classes must be defined
85 // in the file PrimitiveClasses.xml and the .cpp files must also be
86 // added to CMakeList.txt.
87 //
88 // Some requirements for each of the classes.
89 //
90 // a) exported classes.
91 // - are subclasses of RexxObject
92 // - implement a flatten() method so that they can be saved.
93 // - the class must implement a createInstance() static method
94 // - the class must define a classInstance static variable that
95 // will hold a reference to the class object for the type.
96 // - All methods visible to the Rexx programmer are defined in Setup.cpp
97 // - All assignments to instance variables within the class after
98 // initial creation must use setField() to handle old-to-new memory
99 // references.
100 //
101 // b) internal classes
102 // - are subclasses of RexxInternalObject
103 // - also implement a flatten method
104 // - do not have an associated class object or external methods.
105 // - All assignments to instance variables within the class after
106 // initial creation must use setField() to handle old-to-new memory
107 // references.
108 //
109 // c) transient classes
110 // - are also subclasses of RexxInternalObject
111 // - do not require a flatten method.
112 // - Since transient classes are never included in the saved
113 // image, they do not need to use setField() for instance variable
114 // assignments.
115 
116 // Steps to adding a new class
117 // 1) Create the .hpp file and .cpp file. This does not need to be
118 // one-to-one. An hpp file can define multiple classes, but remember that
119 // each class you defined will require the additional steps.
120 //
121 // 2) Add the class definition to the appropriate section of
122 // PrimitiveClasses.xml. The build uses this information to generate
123 // header files and .cpp files that are part of the build.
124 //
125 // 3) Add the new .cpp files to CMakeLists.txt
126 //
127 // 4) If this is an exported class,
128 // a) add a define below mapping the TheXxxxxxXxxxxxxClass to
129 // the static variable holding the class object.
130 // b) add a call to the createInstance() method for the new class
131 // object in Setup.cpp
132 // c) define the exported methods for the class in Setup.cpp.
133 // d) The method mapping table in CppCode.cpp needs to be updated
134 // with all new native code methods exported by the class.
135 // e) Implement a newRexx method with the following signature
136 // that will be the NEW method that is called to create new
137 // instances:
138 //
139 // RexxObject *newRexx(RexxObject **args, size_t argc);
140 //
141 // f) add a RESTORE_CLASS() macro call for the class in RexxMemory::restore()
142 // so that the TheXxxxxxXxxxxxxClass variable gets restored when the image
143 // is loaded.
144 
145 // Steps for writing a new class:
146 // All objects allocated from the object heap must implement
147 // some common features:
148 //
149 // 1) Are subclasses of either RexxInternalObject or RexxObject (only for
150 // exported classes.
151 // 2) Define a new and delete method as part of the class definition.
152 //
153 // void *operator new(size_t);
154 // inline void operator delete(void *) { ; }
155 //
156 // 3) The new method should look like this:
157 //
158 // void *VariableReferenceOp::operator new(size_t size)
159 // {
160 // return new_object(size, T_VariableReferenceOp);
161 // }
162 //
163 // where the T_XxxxxxXxxxxxxXxxxxxx symbol is the class
164 // identifier that is automatically generated from the name
165 // given on PrimitiveClass.xml
166 //
167 // 4) The new class must implement live() and liveGeneral() methods
168 // to interact with garbage collection. These methods look like
169 // this:
170 //
171 //
172 // /**
173 // * Perform garbage collection on a live object.
174 // *
175 // * @param liveMark The current live mark.
176 // */
177 // void VariableReferenceOp::live(size_t liveMark)
178 // {
179 // memory_mark(variable);
180 // }
181 //
182 //
183 // /**
184 // * Perform generalized live marking on an object. This is
185 // * used when mark-and-sweep processing is needed for purposes
186 // * other than garbage collection.
187 // *
188 // * @param reason The reason for the marking call.
189 // */
190 // void VariableReferenceOp::liveGeneral(MarkReason reason)
191 // {
192 // memory_mark_general(variable);
193 // }
194 //
195 // The live methods must mark each of the object variables that
196 // are references to other Rexx objects (of all types). Note that
197 // each class must also mark the symbols defined by the superclass.
198 // In particular, exported classes must mark the symbol "objectVariables"
199 // inherited from the RexxObject class.
200 //
201 // 5) Exported (a) and internal (b) classes must implement a flatten method.
202 //
203 //
204 // /**
205 // * Flatten a source object.
206 // *
207 // * @param envelope The envelope that will hold the flattened object.
208 // */
209 // void VariableReferenceOp::flatten(Envelope *envelope)
210 // {
211 // setUpFlatten(VariableReferenceOp)
212 //
213 // flattenRef(variable);
214 //
215 // cleanUpFlatten
216 // }
217 //
218 // Where the name specified in the setUpFlatten() macro matches name of
219 // the class.
220 //
221 // the garbage collection methods are crucial to proper functioning of the
222 // interpreter. A lot of crashes during development can be traced to failure
223 // in updating the GC methods when a new field is added to the class.
224 //
225 // 6) define the special constructor for the class. This constructor is used to
226 // the virtual function table needed to restore classes during unflattening
227 // operations. This special constructor looks like this:
228 //
229 // inline VariableReference(RESTORETYPE restoreType) { ; };
230 //
