313 // Object Reference Assignment
314 // OrefSet handles reference assignment for situations where an
315 // object exists in the oldspace (rexx image) area and the fields is being updated
316 // to point to an object in the normal Rexx heap. Since oldspace objects do
317 // not participate in the mark-and-sweep operation, we need to keep track of these
318 // references in a special table.
319 //
320 // OrefSet (or the setField() shorter version) needs to be used to set values in any object that
321 // a) might be part of the saved imaged (transient objects like the LanguageParser, RexxActivation,
322 // and Activity are examples of classes that are not...any class that is visible to the Rexx programmer
323 // are classes that will be part of the image, as well as any of the instruction/expresson objects
324 // created by the LanguageParser). Note that as a general rule, fields that are set in an object's constructor
325 // do not need this...the object, by definition, is being newly created and cannot be part of the saved image.
326 // Other notible exceptions are the instruction/expression objects. These object, once created, are immutable.
327 // Therefore, any fields that are set in these objects can only occur while a program is getting translated. Once
328 // the translation is complete, all of the references are set and these can be safely included in the image
329 // without needing to worry about oldspace issues. If you are uncertain how a given set should be happen,
330 // use OrefSet(). It is never an error to use in places where it is not required, but it certainly can be an
331 // error to use in places where it is required.
332 
333 // Important note: In the original version, this was a macro. However, as a macro, if v was an expression,
334 // it ended up getting evaluated twice. The first evaluation got added to the new2old table, the second
335 // was the value assigned to the object. This had two bad side effects: 1) the value stored the object is
336 // not protected from garbage collection and 2) the object added to the old2new table will never get removed,
337 // creating a memory leak. Fortunately, this is a rare situation, having shown up just once in 30 years.
338 
339 inline void OrefSet(RexxInternalObject *o, RexxInternalObject *&r, RexxInternalObject *v)
340 {
341  if (o->isOldSpace())
342  {
343  memoryObject.setOref(r, v);
344  }
345  r = v;
346 }
347 
348 // short cut version of OrefSet(). 99% of the uses specify this as the object pointer...this version
349 // saves a little typing :-)
350 #define setField(r, v) OrefSet(this, (RexxInternalObject *&)this->r, v)
351 // OK, I'm lazy...a version of this for nulling out a field.
352 #define clearField(r) setField(r, OREF_NULL)
353 
354 // cleaner version for setting in another object
355 #define setOtherField(o, r, v) OrefSet(o, (RexxInternalObject *&)o->r, v)
