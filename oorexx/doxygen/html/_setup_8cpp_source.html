400 // Notes on defining external methods. The defined methods create a mapping from
401 // a Rexx method name to a C++ method that implements a method. These are created
402 // via function pointers that are stored in the CPPCode objects backing the object
403 // method. The following conditions must be met for an method to be used:
404 //
405 // 1) The method name must be unique on the class (i.e., no multiple methods with
406 // the same name and different signatures.
407 // 2) With the exception of methods define with A_COUNT (see below), all method
408 // arguments must be RexxObject types and must have a return value that is also a
409 // RexxObject type. If the method does not have a return value, the method needs
410 // to return OREF_NULL.
411 // 3) When the method is defined, it identifies the maximum number of arguments it
412 // can be called with. This needs to match the number of arguments in the C++ method
413 // signature. This dispatcher will sort out omitted arguments and pass OREF_NULL
414 // for those. The target method needs to sort out required vs. optional arguments.
415 // 4) For methods with an open-ended number of arguments, A_COUNT is specified. These
416 // methods must have an signature of "RexxObject *name(RexxObject **args, size_t argCount)"
417 // to handing being passed a variable-sized argument list.
418 // 5) The target method can be defined as virtual, but it is called directory using
419 // the function pointer and not the object virtual function table. To get virtual function
420 // dynamics, create a stub method to act as a bridge between the Rexx method call and
421 // the virtual method call.
422 // 6) These methods are stored in the rexx.img file and they need to reestablish
423 // their method pointers at startup time. This is done via a table in CPPCode that
424 // allows a mapping to be created between the function pointer and a numeric index.
425 // When you add a new method to the setup below, you also need to update the CPPCode
426 // table to include the function pointer. A build error will result if the table is
427 // not updated.
428 // 7) A lot of objects like string or the collection classes are used internally in
429 // the interpreter. These typically have a low-level method that performs a function
430 // and a Rexx stub method that does error checking and any needed conversion before
431 // calling the lower level method. These methods by convention have a "Rexx" suffix
432 // in the name.
