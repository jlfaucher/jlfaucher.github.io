◆ removeWaitingActivity()

void ActivityManager::removeWaitingActivity ( Activity *  waitingAct)
static

In certain situations (mostly confined to Windows), we can re-enter the interpreter on a thread that is currently on the dispatch queue. If that activity gets dispatched, then everything blocks because the activity never wakes up until the nested thread returns. In that situation, we need to ensure that the original waiting activity is removed from the dispatch queue until it is the top activity on the stack.

Parameters
waitingActThe activity to remove from the queue.
