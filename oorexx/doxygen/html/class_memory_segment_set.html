◆ adjustMemorySize()

void MemorySegmentSet::adjustMemorySize ( )
protected

Decide if we should add segments from the segment set based on the current state of the heap immediately following a sweep. We want to avoid thrashing the garbage collector because of chronic low memory conditions. If we wait until we get an actual allocation failure, we will end up in a state where we're constantly running a mark-and-sweep to reclaim just a tiny amount of storage, and then repeat. We will try to keep the heap with appropriate free space limits.

If we're starting to run full, then we should expand the heap to prevent recycling the garbage collector to reclaim minimal amounts of storage that is immediately reused, causing another GC cycle.
